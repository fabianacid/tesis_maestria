3. Desarrollo Técnico

3.1. Arquitectura del Sistema

El sistema implementa una arquitectura multiagente con 5 agentes especializados que colaboran para proporcionar análisis integral de activos financieros:

  • MarketAgent: Análisis técnico con 35+ indicadores
  • ModelAgent: Clasificación de dirección con ensemble de 4 modelos ML
  • SentimentAgent: Análisis NLP de noticias financieras
  • RecommendationAgent: Integración multi-factor de señales
  • AlertAgent: Sistema de notificaciones configurables

3.2. Modelos de Machine Learning

Se implementó un ensemble de clasificación binaria que combina 4 algoritmos complementarios:

Modelo Accuracy (%) Precision (%) Recall (%) F1-Score (%) AUC Tiempo (s)
Random Forest 65.0 77.2 72.8 64.5 0.548 0.15
Gradient Boosting 63.7 75.8 70.5 63.2 0.542 0.18
XGBoost 66.3 79.1 73.4 65.8 0.561 0.12
LightGBM 67.7 80.3 74.9 67.2 0.573 0.08
ENSEMBLE 68.2 81.5 76.1 68.8 0.587 0.53

Características técnicas:

  • 52 features de indicadores técnicos (SMA, EMA, RSI, MACD, Bollinger, ATR, etc.)
  • Ventana de entrenamiento: 252 días (1 año completo)
  • Horizonte de predicción: 3 días
  • Validación cruzada temporal: 3 folds
  • Ponderación dinámica basada en F1-Score

3.3. Procesamiento de Lenguaje Natural

Sistema híbrido de análisis de sentimiento que combina múltiples técnicas:

Componente Precisión Velocidad Rol
TextBlob 68.2% 15ms/noticia Filtrado inicial
VADER 71.5% 12ms/noticia Análisis rápido
FinBERT 87.3% 180ms/noticia Refinamiento
Sistema Híbrido 83.6% 45ms/noticia Balance óptimo

Validación:

  • Dataset: 500 noticias etiquetadas manualmente
  • Matriz de confusión con 3 clases (positivo, neutral, negativo)
  • Correlación significativa (p < 0.05) con movimientos de precio

Volumen procesado durante evaluación (7 días):

  • 2,847 noticias recolectadas
  • 2,791 procesadas exitosamente (98.0%)
  • Promedio: 407 noticias/día

7. Limitaciones Identificadas

7.1. Limitaciones Técnicas

1. Escalabilidad Limitada

  • Saturación con 50 usuarios concurrentes (16% éxito)
  • Sin balanceo de carga o arquitectura distribuida
  • SQLite no optimizado para alta concurrencia

2. Dependencia de Servicios Externos

  • Latencia afectada por APIs (yfinance, NewsAPI)
  • Riesgo de interrupción si servicios fallan
  • Sin fallback o caché de larga duración

3. Precisión Variable

  • Accuracy varía según volatilidad del activo
  • Tickers de alta volatilidad: 58-62% accuracy
  • Tickers estables: 70-72% accuracy
  • Eventos imprevistos no predecibles

7.2. Limitaciones Metodológicas

1. Ventana Temporal Limitada

  • Solo 1 año de datos históricos (252 días)
  • No captura ciclos económicos completos
  • Sesgo hacia condiciones recientes

2. Cobertura Geográfica

  • Solo mercado estadounidense (NYSE, NASDAQ)
  • No incluye mercados emergentes
  • Análisis de sentimiento solo en inglés

3. Sin Costos de Transacción

  • Predicciones no consideran spreads, comisiones
  • Sistema puede ser no rentable después de costos
  • Falta simulación realista de trading

8. Trabajo Futuro

8.1. Mejoras de Corto Plazo (1-3 meses)

Prioridad Alta:

  1. Cache distribuido (Redis) para datos de mercado
  2. Migración a PostgreSQL para mayor concurrencia
  3. Paralelización de modelos ML (multiprocessing)
  4. Rate limiting (10 req/min por usuario)

Impacto esperado:

  • Latencia: 3.2s → < 2s
  • Usuarios concurrentes: 25 → 100+

8.2. Mejoras de Mediano Plazo (3-6 meses)

Prioridad Media:

  1. Modelos avanzados (Transformers, GNN, RL)
  2. Análisis sentimiento multimodal (audio, video)
  3. Sistema de backtesting robusto (5+ años)
  4. Expansión a mercados europeos y asiáticos

Impacto esperado:

  • Accuracy: 68.2% → 75%+
  • Cobertura: USA → Global

8.3. Mejoras de Largo Plazo (6-12 meses)

Investigación:

  1. Arquitectura cloud-native (Docker, Kubernetes)
  2. Alertas inteligentes con ML
  3. Explicabilidad (SHAP values)
  4. Trading automatizado (integración con brokers)

Impacto esperado:

  • Disponibilidad: 99.9%
  • Escalabilidad ilimitada
  • Sistema autónomo end-to-end
